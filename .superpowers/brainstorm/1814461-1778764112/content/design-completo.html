🗂️ Feature 1 — Gestão de Planos

Backend — novos endpoints
GET /api/v1/planos # público (bot+UI)
POST /api/v1/planos # criar plano (ADMIN)
PATCH /api/v1/planos/{id} # editar (ADMIN)
DELETE /api/v1/planos/{id} # excluir (ADMIN)
Schema por plano
nome: string
preco: float
velocidade: string # "35MB"
extras: string[]
descricao: string # novo
ativo: bool # novo
destaque: bool # novo
Armazenamento
Continua como JSONB na tabela config (chave "planos"). Os novos endpoints são wrappers que leem/escrevem o array completo atomicamente. Não precisa migração de banco.
Frontend — nova página
/configuracoes/planos
Planos de Internet
Essencial ativo
35MB · R$110
Internet fibra básica
Plus ⭐ destaque
55MB · R$130
Com IPTV incluso
Modal de edição com campos: nome, preço, velocidade, descrição, extras (chips removíveis + campo para adicionar), toggles ativo/destaque. Bot só apresenta planos com ativo=true. Plano com destaque=true recebe borda destacada.

📄 Feature 2 — PDF de OS

Backend
GET /api/v1/os/{id}/pdf
→ retorna PDF (application/pdf)
POST /api/v1/os/{id}/enviar-pdf-tecnico
→ gera PDF + envia ao técnico via Evolution
Geração (WeasyPrint)
1. Template HTML Jinja2 renderiza dados da OS
2. WeasyPrint converte HTML+CSS → PDF em memória
3. Fotos lidas do filesystem e embutidas como base64 na pág 2
4. PDF retornado como StreamingResponse ou salvo em /tmp para envio
Página 1 do PDF inclui
Header Ondeline · Código OS · Nome + tel cliente · Nome + tel técnico · Endereço · Problema · Status · Agendamento · Data conclusão · CSAT (se disponível)
Página 2 (fotos)
Fotos em grid 2 colunas com timestamp de cada. Se não houver fotos, página 2 omitida.
Frontend
Na página de detalhe da OS (os-detail.tsx), dois botões novos:
Detalhe da OS — OS-ABC123
Problema: Internet caindo
Carlos Silva · Rua das Flores, 123
"Baixar PDF" abre nova aba com o PDF. "Enviar PDF ao Técnico" faz POST e exibe toast de confirmação. Botões disponíveis para ADMIN e ATENDENTE.

📊 Feature 3 — Ranking de Técnicos

Backend — novos endpoints
GET /api/v1/metricas/tecnicos
?mes=2026-05 (default: mês atual)
GET /api/v1/metricas/tecnicos/export
?mes=2026-05 → CSV para download
Response por técnico
tecnico_id: UUID
nome: string
os_concluidas: int
csat_avg: float | null
tempo_medio_min: int | null
ultima_os_em: datetime | null
Query agrupa OS por tecnico_id WHERE status=CONCLUIDA E concluida_em no mês. Ordenado por os_concluidas DESC.
Frontend — /tecnicos/ranking
Ranking de Técnicos

Nova rota no menu lateral abaixo de "Configurações". Exportação CSV inclui: nome, OS concluídas, CSAT médio, tempo médio — pronto para copiar numa planilha de pagamento.
